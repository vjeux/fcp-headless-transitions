## What I changed

1. **`fct/swarm/pool.py` — stall detection + PGID-wide reap.** Added `_slot_stalled(slot_info, task_id)` and `_kill_slot_pg(slot)`. Every scheduler pass, a slot that (a) has been running ≥ `STALL_MIN` (20m), (b) whose log hasn't grown for ≥ `STALL_QUIET_MIN` (15m), and (c) has no SWARM_RESULT is now killed — and killed by process-group (`ps -o pgid` → `pgrep -g` → `kill -TERM`, then SIGKILL after 2s) so the MCP orphans that were holding tee's pipe open die with it. The slot then re-enters the queue on the next tick. This is the direct fix for the T-B3/T-F1/T-E2 wedge above. Gate: harness-only, no engine touched.

2. **`fct/swarm/push_helper.sh` — belt-and-suspenders `.git` protection.** Added `--filter='protect /.git'` and `--filter='protect /.git/**'` ahead of the existing excludes. Rsync's `protect` rule is a hard "never delete this on destination" instruction that operates outside the include/exclude machinery, so it can't be tripped by whatever race/metadata edge produced the T-B1 harvest error. The existing excludes stay in place; this is purely defensive.

3. **`fct/swarm/reflect.py` — surface wedged slots in metrics.** `gather_metrics` now walks the same "old + tiny + no result" heuristic and emits a `## Wedged agent logs` table in the snapshot. Future reflections don't have to eyeball ps + tmux state — the pattern shows up in the metrics table directly.
