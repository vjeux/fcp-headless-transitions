Shared HGColorMatrix over-1.0 clamp — overflow-coupling analysis (2026-07-23)
================================================================================
Affects: PAEContrast, PAEBrightness, PAEChannelMixer_clip, PAEHSVAdjust_combined
(all when one channel's pre-clip WS value exceeds 1.0 on a SATURATED input).
The gray/in-range legs are VERIFIED; the divergence is ONLY the over-1.0 highlight
redistribution. This file records the mechanism's measured shape (NEW evidence toward
the eventual GPU/Metal tile-readback disasm — still the documented true blocker).

SETUP: Contrast in the gamma-1.958 WS about 0.5 pivot (VERIFIED-exact on gray, 0.79 lvl):
  ws = (in/255)^0.51117 ; pre = (ws - 0.5)*c + 0.5 ; then FCP's clamp/redistribute ; decode.

KEY MEASUREMENT — the clipped channel is clamped to exactly 1.0; the OTHER channels are
LIFTED. Lift vs the clipped channel's overflow (pre-1.0 excess):

  input        c    overflow   lift_on_others      lift/overflow
  (50,50,200) 2.0    0.266     [0.266, 0.266]      [1.00, 1.00]   <- lift == overflow EXACTLY
  (50,200,50) 2.0    0.266     [0.441, 0.356]      [1.65, 1.33]   <- asymmetric, > overflow
  (200,50,50) 2.0    0.266     [0.313, 0.293]      [1.18, 1.10]
  (50,50,200) 1.5    0.075     [0.260, 0.260]      [3.48, 3.48]   <- lift >> overflow at c=1.5
  (50,200,50) 1.5    0.075     [0.381, 0.317]      [5.09, 4.24]
  (200,50,50) 1.5    0.075     [0.290, 0.275]      [3.87, 3.68]

FINDINGS:
 1. The clipped channel always resolves to 1.0 (WS) = 255 (code). Simple hard clamp there.
 2. For the fully-symmetric (50,50,200) at c=2.0 the lift on both equal channels EQUALS the
    overflow exactly (0.266) — consistent with "add the clipped excess to the other channels".
 3. But that simple additive model FAILS elsewhere: the lift is ASYMMETRIC between the two
    other channels (0.441 vs 0.356 for (50,200,50)) and the lift/overflow ratio is NOT 1 at
    c=1.5 (3.5-5x). So the excess is NOT simply added; the lift depends on each channel's own
    pre-clip value AND on c in a way no per-channel closed form reproduces.
 4. RULED OUT this session (added to the prior 8): (a) equal-headroom redistribution,
    (b) luma-preservation desaturation, (c) luma-contrast expansion (scale chroma+luma about
    0.5), (d) scale-whole-vector-so-max=1. None match; best residual ~50-133 lvl.

CONCLUSION: the over-1.0 behaviour is a CROSS-CHANNEL redistribution whose coefficients are
not a pointwise function of (pre-clip vector, c). It matches the long-standing conclusion that
this is an OZ tile-conversion / Metal readback effect (ExtendedLinearSRGB -> display) that
must be read from the GPU shader, NOT fit from transfer probes. All affected nodes stay
CHARACTERIZED (VERIFIED on their in-gamut regions). decode-don't-fit holds.
