# fct engine audit — end-to-end transition correctness (2026-07-24)

## What this corrects
Node-boundary PARITY (fct/parity) VERIFIED 53 isolated FCP functions (colour transfers, blur
kernels) bit-for-bit. That is NECESSARY but NOT SUFFICIENT: it does not prove the whole
transitions render correctly. This audit measures the ACTUAL 65 shipped transitions,
engine-vs-headless-FCP, per frame.

## Honest headline
- 65 transitions, mean 17.83 dB, median 16.60 dB engine-vs-headless.
- Only 6/65 are "ok" (>25 dB). 36/65 are bad or broken (<18 dB).
- The node-boundary VERIFIED colour/blur math is fine; the FAILURES are in
  RIG / MOVEMENT / 3D-TRANSFORM / COMPOSITING / OUTPUT-GEOMETRY — the parts the parity
  harness never exercised.

## Method
Per transition, scored engine-vs-headless PSNR at frame 0 (rest=A), mid, last (rest=B).
The endpoint-vs-midpoint signature classifies the root cause:
- endpoints HIGH, mid LOW  -> MOTION bug (rest states right, the animation between is wrong)
- endpoints LOW            -> STRUCTURAL bug (wrong geometry/layer/composite even at rest)
- output dims != 1920x1080 -> DIM/aspect bug
- all moderate             -> mid/resample (kernel ok, resampling/settle ceiling)

## Buckets
| class | n | mean dB | meaning |
|-------|---|---------|---------|
| STRUCTURAL | 31 | 14.4 | wrong geometry/layer/composite/rig even at endpoints — TOP PRIORITY |
| MOTION | 13 | 19.1 | endpoints correct, the movement/timing between is wrong |
| mid/resample | 15 | 22.2 | kernel correct, mid-transition softening/settle ceiling |
| ENDPOINT | 5 | 22.8 | one endpoint (A or B) wrong — layer visibility/timing at rest |
| DIM/aspect | 1 | 15.4 | engine renders wrong canvas size |

## The minimize tool IS the right instrument
fct/minimize.py (ddmin delta-debugger) reduces a broken .motr to the MINIMAL subtree that
still makes engine diverge from REAL FCP-headless, giving the exact node to fix. Already done
for 12 cases (e.g. Stylized__Lower 153->3 nodes, Video_Wall 46->4, Concentric 140->11).
CAVEAT: some minimize runs ABORT at 99 dB headless (Smear, Squares) — meaning the discrepancy
is engine-vs-GUI (colour management), NOT engine-vs-FCP; those are a separate class.

## IN-DEPTH DIAGNOSIS: Wipes__Diagonal (worst minimized case, worst frame 2.60 dB) — 2026-07-24
Minimized 6-node repro: "Transition Diagonal" layer = {"Gradient and background" group (a
light-gray gradient-fill shape "Background") + "Animated mask" (feathered closed bezier, Feather=300,
Aspect 1.5)} composited over "Transition Drop Zones" (image A). Emitters + Transition B stripped.

DECODED FCP behavior (headless, per-corner sampling):
- f0-4: image A held (mean 97).
- f5-14: a light-gray gradient WASHES IN diagonally from TOP-LEFT, reaching full-frame uniform ~195.
- f16-18: a SECOND, DARK wavefront sweeps in from TOP-LEFT over the gray (TL 195→114→0 while BR still 195).
- f20-23: fully BLACK (all corners 0).
=> FCP "Diagonal" is a MOVING REVEAL BAND (two diagonal wavefronts): a leading edge reveals the gray
   gradient over A, a trailing edge conceals it to black. NOT a monotonic write-on envelope.

ENGINE behavior (decoded via FCT_WON_DEBUG write-on coverage trace):
- The engine models the "Animated mask" as a MONOTONIC WRITE-ON envelope (union of the feathered
  quad rasterized at K=12 sub-times over [0,curveTime], per-pixel max). Coverage caps at 85.6%
  (a central vertical band; sepia A remains on L/R edges) and HOLDS — it never reaches 100% and
  never sweeps to black. So engine mean freezes at ~175 while FCP goes 195→0.
TWO ROOT DEFECTS:
  (1) COVERAGE CAP: the finite feathered quad (~1000px + 300 feather) swept along the diagonal
      Position path [(-1046,-2129)@0.30s → (+2105,+1871)@1.07s] leaves the opposite corners
      uncovered; the write-on union tops out at 85.6%. FCP covers the whole frame.
  (2) WRONG MODEL: FCP is a moving band (reveal-then-conceal to black), not monotonic write-on.
      The engine's monotonic-max hold can never reproduce the black exit sweep.
NEXT: replace/augment the write-on model for this mask with the true moving-band reveal (leading
edge reveals gradient, trailing edge conceals) so it (a) reaches full coverage and (b) exits to
black. Verify the Position sweep path + quad size reproduce both wavefront timings. This is a
compositor-model fix (engine/src/compositor + evaluator write-on), not a constant tweak.
